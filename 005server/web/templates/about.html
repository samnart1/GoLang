Features implemented

  • http server: build with go's net/http package
  • routing: custom route handling with serveMux
  • middleware: logging, cors, and recovery middleware
  • templates: html template rendering with data binding
  • json apis: restful endpoints with json responses
  • static files: css, js, and image serving
  • configuration: environment-based config
  • logging: structured logging with different levels
  • graceful shutdown proper server lifecycle management

project structure

this project follow's structure is as bellow

  • cmd - CLI commands and application entry points
  • internal - private application code
  • pkg - public reusable packages
  • web - static assets and templates
